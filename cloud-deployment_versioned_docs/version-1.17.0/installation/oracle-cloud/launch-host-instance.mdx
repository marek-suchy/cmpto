This guide explains how to launch an Oracle Cloud Infrastructure (OCI) instance for running Live Transcoder.

## Instance Requirements

Live Transcoder requires an Nvidia GPU-equipped instance. The recommended starting shape is **VM.GPU2.1**, which provides:
- Single Nvidia Tesla P100 GPU
- 12 OCPUs

:::warning
Live Transcoder does not yet support Ada or Hopper GPU architectures on OCI.
:::

## Resource Planning

CPU requirements depend on your stream specifications:

| Stream Type | OCPUs Required |
|---|---|
| 1080p H.264 4:2:0 8-bit | ~2 OCPUs per stream |
| 1080p H.264 4:2:2 10-bit | ~16 OCPUs per stream |

If CPU utilization exceeds **80%**, increase the instance size.

## 1. Check and Request GPU Quotas

Oracle Cloud enforces limits on GPU instances per region and availability domain.

1. Navigate to **Governance & Administration > Limits, Quotas and Usage**.
2. Search for GPU-related limits in your target region and availability domain.
3. Submit a limit increase request if needed before proceeding.

## 2. Launch the Instance

1. Navigate to **Compute > Instances > Create Instance**.
2. Select your desired region and availability domain.
3. Under **Shape**, select **VM.GPU2.1** (or another GPU shape as needed).
4. Select **Ubuntu 24.04** as the operating system.

## 3. Storage

Allocate a minimum of **100 GiB** for the boot volume. Add additional storage if you plan to use HLS outputs.

:::warning
Insufficient disk space can compromise diagnostic logging capabilities, which are essential for troubleshooting.
:::

## 4. Configure Network Security Groups

Create or update a Network Security Group (NSG) with the following ingress rules:

| Port(s) | Protocol | Purpose |
|---|---|---|
| 22 | TCP | SSH |
| 80 | TCP | HTTP (Web UI, HLS) |
| 443 | TCP | HTTPS |
| 5353–7999 | TCP/UDP | NDI |
| 1935 | TCP | RTMP input |
