# Twenty-One Encoder User Interface

Comprimato Twenty-One Encoder is **enriched with an easy-to-use web-based user interface which is the primary environment users should use**. By default, the Twenty-one encoder starts this UI with every boot. Here, the main configuration takes place as well as monitoring tools present the data.

There are two ways to access the UI:

Locally - being physically present within the server location and using kiosk mode (in case there is a monitor connected to the machine).

Remotely - using the assigned administration interface IP address in your web browser to remotely connect to administration.

### Demo licenses

Comprimato offers demo licenses for testing purposes for a limited time. The demo license may be loaded with the Load license file button (right) and will allow unlimited functionality for streams. If you need more time for testing your use cases, you may contact Comprimato support and request another demo license.

If a license is granted, the status shows “Unlimited time” or the time for which the pipeline can produce an output, according to loaded license files (see section 5.3). The status will be for example:

License: Unlimited time – a pipeline may produce an output for unlimited time.

License: Till 2021-02-21 23:59:59 – a pipeline will produce an output until the displayed time point is passed (please note that for your convenience, the status shows the time and date according to your computer (i.e., browser) time, not the twenty-one encoder system time, which may be different).

If the pipeline is not allowed to produce an output, the license status reads “Blocked” or “Unavailable” and is shown red. Twenty-one encoder will process the pipeline as usual, the only difference is that the output will not be sent to defined output sinks. This feature will help you verify your environment abilities (e. g. system load limits, input stream compliance and others) even if you do not have enough licenses yet. For that reason, the input and output stream status indicator may be green even if the license status is not valid.

Possible invalid license statuses are:

License: Blocked – there are licenses for the stream loaded in the system, but the count of streams would exceed the licensed number (other streams have exhausted the available licenses)

License: Unavailable – the stream needs a license that is not loaded (the stream contains a feature of a newer version of Twenty-one encoder which isn’t included in any loaded license)

### Twenty-One Encoder appliance licenses

Twenty-One Encoder appliance will contain a perpetual license for the end user. This license will be unlimited regarding duration and pipeline count. However some features, such as JPEG-XS will be restricted unless included in the license. Please contact Comprimato Support or your reseller for more information.

#### Low latency

If the use case demands the lowest achievable latency, choose the Low latency configuration in Advanced Settings management by selecting the file low_latency.ini. If the output is in Transport Stream form, Transcoder will change the bitrate from a constant value to a variable one (from CBR to VBR), and Transcoder tries to optimize its internal configuration to get as close as possible to the lowest obtainable latency.

You can improve this for outputs using H.26x codecs by configuring the pipeline to have 0 B frames on the output, and if you can adjust the input stream, it is best to not include any B frames there, too, for latency purposes. Note, B frames cannot be used when using low latency mode. If used in combination with SRT, you should also set the SRT latency to a low value on the receiver’s end. Please measure the ping from sender to receiver, the minimum SRT latency must be at least double this, to allow for successful re-transmission for lost packets.

#### Manual Low Latency Adjustments

Latency addition is the offset applied to the PTS for reliable transmission to the receiving decoder. The offset is calculated by the overall configuration of the pipeline. The latency addition field allows for the manual adjustment of this offset. This field is available on all MPEG-TS outputs within Twenty-One Encoder.

It is recommended to obtain a baseline figure before adjusting the latency addition manually using the Live Transcoder REST API:

Use the following GET command to obtain the pipeline statistics when a pipeline is active: http://hostIPaddress/api/pipeline/0/state

Look for the field “transcodingLatency”

Transcoding Latency is the real time figure of the pipeline, you cannot set the latency addition lower than this.

It is recommended to add 40-50ms+ of the transcoding latency in the Latency Addition field and restart the pipeline

It is best to test using complex content, as this will need more time to encode. If you notice frame drop errors then the Latency Addition will need to be increased.

### Reinstallation

#### Reinstallation from the UI

If your server behaves strangely, underperforms, or has a broken configuration, you can reinstall it. The reinstallation feature makes a clean installation of the currently booted version. It keeps only licenses; everything else turns back to default **(including passwords)**.

You can start the Reinstallation by clicking the Perform reinstallation button on the Version Management page.

*Figure: Reinstallation button*

Once changes are applied, you are presented with the result and the server reboots itself. The following boot may take longer, and the server may reboot itself several times to finish the process. When the Reinstallation is completed, the machine will boot up normally, allowing the user to login into UI.

#### Reinstallation from the bootloader menu

If your server can’t boot successfully, or you forgot the UI password, you can perform a Reinstallation from the bootloader menu during boot-up. Select the “> REINSTALL LATEST” item with arrows and press Enter on your keyboard.

*Figure: Bootloader screen with the “> Reinstall latest” item*

Then the server boots into a text environment where it asks for confirmation of the reinstallation process to not start it by mistake.

*Figure: Reinstallation confirmation*

After the Reinstallation finishes, you are presented with its result, and the server reboots itself. The following boot may take longer, and the server may reboot itself several times to finish the process. Once the Reinstallation is completed, the machine will boot up normally, allowing the user to login into UI.

## MPEG-TS Hitless Protection Switching

Hitless protection switching is a method to ensure safe transmission over an unstable link. The same transport stream is being sent via two different paths. The receiver then realigns the two streams and can create a single output from it. This way if an error occurs during the transmission on one path, the receiver can obtain the required data from the other one. For more details, please see SMPTE 2022-7.

Twenty-one encoder can both receive and generate streams with Hitless Protection Switching. In order to receive this kind of stream, the server needs to receive the identical stream from two sources via a single entry point (one network interface). The input configuration is identical as if it was just a regular stream.

Twenty-one encoder needs two network interfaces configured for data transmission to transmit a stream using the Hitless Protection Switching. Once you configure the output stream options, you then have to add another sink (under Output stream sink, using the “Configure sinks” option), which will have identical IP:port combination but will use a different network interface.

Twenty-one encoder also supports SMPTE 2110 2022-7 which is explained in section 4.12
